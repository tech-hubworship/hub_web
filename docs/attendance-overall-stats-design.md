## 1. 엑셀 시트 구조 요약

| 영역 | 행/열 | 내용 |
|------|--------|------|
| **지각비 기준** | 1~5행 | 시간 구간별 금액 (9:31~9:40 → 1,000원 … 11:11~ → 4,000원), 무단결석 5,000원, 보고서 유형 등 |
| **분기 납부현황** | 7~8행 | 1분기 313,000 / 2분기 … / 4분기, 주차별 날짜 헤더 (11월 8일, 11월 15일 …) |
| **메인 그리드** | 12행~ | 행: 그룹 → 다락방 → 그룹장/다락방장/회원명, 열: **주차(날짜)** → 셀값: 지각비(원), 0, -(결석/미기록) |
| **소계 행** | 일부 | 다락방별 출석 인원 수 (6, 7, 5 …) |

- **행**: 그룹명(믿음, 기쁨, 순종 …) → 다락방 → (그룹장/다락방장 표시) → 회원 이름.
- **열**: 주차별 **일요일 또는 OD 기준일** (11/8, 11/15, 11/22 …).
- **셀**: 해당 주 해당 사람의 지각비(숫자), 0(정상/지각 0원), -(미출석·결석·예외 등).

## 2. 현재 DB로 표현 가능 여부

| 시트 항목 | DB/코드 매핑 | 비고 |
|-----------|----------------|------|
| 그룹·다락방·이름 | `profiles` (group_id, cell_id) + `attendance_od_targets` (name) | hub_groups, hub_cells로 명칭 조회 |
| 그룹장/다락방장 | `attendance_od_targets.is_group_leader`, `is_cell_leader` | 이미 있음 |
| 주차별 날짜 | `weekly_attendance.week_date` | 날짜 단위 1:1 |
| 셀값(지각비/0/-) | `weekly_attendance.late_fee`, `status`, `is_excused` | present/late→금액, unexcused_absence→5000, excused→예외(-) |
| 분기 납부현황 | `weekly_attendance.late_fee` + `late_fee_settlements` | 기간별 sum, 정산 반영 가능 |

**결론**: 별도 테이블 없이 **기존 DB만으로** 시트와 동일한 “전체 통계” 그리드를 만들 수 있음.

## 3. 전체 통계 화면 구상 (한눈에 보기)

### 3.1 레이아웃

```
┌─────────────────────────────────────────────────────────────────────────┐
│ [기간 선택] 2024-01-01 ~ 2024-03-31  [분기] 1분기 ▼  [조회]              │
├─────────────────────────────────────────────────────────────────────────┤
│ 지각비 기준 (요약)  │  9:31~9:40 1천 … 11:11~ 4천  │  무단결석 5천       │
│ 분기 납부현황      │  1분기 313,000  │  2분기 …  │  3분기 …  │  4분기  │
├─────────────────────────────────────────────────────────────────────────┤
│ 그룹    │ 다락방 │ 역할     │ 이름   │ 11/8 │ 11/15 │ 11/22 │ ... (가로 스크롤) │
├─────────┼────────┼──────────┼────────┼──────┼───────┼───────┼───────────────────┤
│ 믿음    │ (그룹장)│          │ 박세리  │  0   │  0    │ 2,000 │                   │
│ 믿음    │ 기쁨   │          │ 구세빈  │  0   │  0    │  0    │                   │
│ 믿음    │ 기쁨   │          │ 이우연  │  0   │  0    │  0    │                   │
│ ...     │        │ (다락방장)│ ...    │  -   │  -    │  0    │                   │
│ 믿음    │ 소계   │          │ (6명)   │  6   │  7    │  5    │  ← 출석 인원 수   │
├─────────┼────────┼──────────┼────────┼──────┼───────┼───────┤                   │
│ 순종    │ (그룹장)│          │ 김세영  │  0   │  0    │ 3,000 │                   │
│ ...     │        │          │        │      │       │       │                   │
└─────────────────────────────────────────────────────────────────────────┘
```

- **상단**: 기간/분기 선택, 지각비 기준 요약, 분기별 납부 합계(선택 시 정산 반영).
- **메인**: 그리드 테이블  
  - **행**: 그룹 → 다락방 → (그룹장/다락방장 표시) → 이름 (OD 명단 + profiles 기준).  
  - **열**: 주차(week_date) 목록. 가로 스크롤로 주차 많아도 한눈에 보이게.
- **셀**: 해당 주 해당 사람  
  - 지각비 금액(1,000 / 2,000 … 5,000),  
  - 0 (출석/지각비 0),  
  - `-` (미출석·결석·예외 등).
- **소계 행**: 다락방별 해당 주 출석 인원 수 (선택 사항).

### 3.2 데이터 흐름

1. **기간 선택**: 예) 2024-01-01 ~ 2024-03-31.
2. **주차 목록 생성**: 해당 기간의 OD 기준 요일(예: 목요일) 또는 “매주 일요일” 리스트 생성 → 그리드 열 헤더.
3. **행 데이터**:  
   - OD 명단(`attendance_od_targets`) + `profiles` (group, cell) → 그룹·다락방·이름·그룹장/다락방장 정렬.  
   - 각 (user_id, week_date)에 대해 `weekly_attendance`에서 `late_fee`, `status`, `is_excused` 조회.  
   - 표시: `is_excused`면 `-`, 아니면 `late_fee` (0이면 0, 5000이면 무단결석 등).
4. **분기 납부현황**:  
   - 1~3월, 4~6월, 7~9월, 10~12월별로 `weekly_attendance.late_fee` 합계.  
   - 필요 시 `late_fee_settlements`로 정산 차감한 “미납/잔액” 표시 가능.

### 3.3 API 제안

- **GET** `/api/admin/attendance/overall-stats?start=2024-01-01&end=2024-03-31`  
  - **응답**:  
    - `weekDates`: string[] (주차 날짜 리스트)  
    - `rows`: { group_name, cell_name, user_id, name, is_group_leader, is_cell_leader, weeklyFees: { [weekDate]: number | null } }[]  
      - `null` 또는 미출석/예외는 프론트에서 `-`로 표시.  
    - `quarterlyTotals`: { quarter: "1", total: 313000 }[] (선택)

구현 시 `attendance/list`와 유사하게 OD 명단 + profiles 조인 후, 기간 내 `weekly_attendance`를 user_id·week_date로 맵핑해 `weeklyFees` 구성하면 됨.

## 4. 이관 정책 (엑셀 → DB)

### 4.1 매핑 전제

- 시트의 **그룹명·다락방명·이름** = DB의 `hub_groups.name`, `hub_cells.name`, `profiles.name` (또는 `attendance_od_targets.name`).  
- 시트의 **주차** = `weekly_attendance.week_date` (YYYY-MM-DD).  
- 시트의 **셀 값** = 해당 주 해당 사람의 `late_fee` (0, 1000, 2000, 3000, 4000, 5000). `-`는 미기록이면 삽입 안 함 또는 별도 플래그.

### 4.2 이관 방식 (택 1 또는 병행)

| 방식 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **A. 수동 재입력** | 과거는 시트 참고만 하고, 이후부터는 시스템에만 입력 | 구현 없음, DB 단순 | 과거 데이터는 시스템에 없음 |
| **B. 엑셀 업로드** | 엑셀/CSV에서 “이름 + 주차(날짜) + 지각비” 컬럼 읽어서 `weekly_attendance` insert/update | 과거 데이터 일괄 반영 | 이름→user_id 매핑 필요, 그룹/다락방은 profiles에 따라 자동 |
| **C. 시트 연동** | Google Sheets API로 읽어서 주기적 동기화 | 시트가 마스터일 때 유리 | API 키·권한, 스키마 호환 유지 필요 |

### 4.3 권장: B (엑셀 업로드) 기준 이관 절차

1. **준비**
   - OD 명단(`attendance_od_targets`) + profiles에 그룹/다락방/이름이 현재 시트와 맞게 정리되어 있는지 확인.
   - 시트에서 “이름”이 한 명만 매칭되도록 동명이인 보정(그룹/다락방으로 구분).

2. **엑셀 형식 (임포트용)**
   - 컬럼: `name`(또는 `이름`), `group_name`, `cell_name`(선택), `week_date`(YYYY-MM-DD), `late_fee`(0~5000).  
   - `-` 또는 빈칸 → 해당 주는 insert 안 하거나, 나중에 “미출석/예외”용 값(예: -1) 규칙 정해서 처리.

3. **임포트 API (추가 제안)**
   - **POST** `/api/admin/attendance/import-weekly`  
   - body: `rows: { name, group_name?, cell_name?, week_date, late_fee }[]`  
   - 로직: name(+ group_name/cell_name)으로 `profiles`/`attendance_od_targets`에서 `user_id` 조회 → `weekly_attendance` upsert (user_id, week_date, category=OD, late_fee, status는 late_fee에 따라 derived).

4. **이관 순서**
   - 1) 그룹/다락방/OD 명단을 시트와 동일하게 맞춤.  
   - 2) 시트 데이터를 위 엑셀 형식으로 추출(수동 또는 스크립트).  
   - 3) 임포트 API 또는 배치 스크립트로 insert.  
   - 4) 전체 통계 화면에서 기간 선택해 시트와 비교 검수.

### 4.4 분기 납부현황 313,000 등

- 시트의 “1분기 지각비 납부현황 313,000”은 **해당 분기 지각비 합계**로 해석 가능.  
- DB: 해당 분기 `weekly_attendance`의 `late_fee` sum.  
- 정산 반영 시: 같은 기간 `late_fee_settlements` sum을 빼서 “미납 잔액”으로 표시할 수 있음 (현재 지각비 관리 화면 로직과 동일).

## 5. 구현 순서 제안

1. **API**  
   - `GET /api/admin/attendance/overall-stats?start=&end=`  
   - 기간 내 주차 목록, OD 명단(그룹·다락방·이름·역할), (user_id, week_date)별 late_fee 맵 반환.

2. **관리자 화면**  
   - “OD 출석 전체 통계” 탭/메뉴 추가.  
   - 기간/분기 선택 → 위 그리드 + 상단 지각비 기준·분기 납부 요약.

3. **이관**  
   - 엑셀 포맷 정의 및 `POST /api/admin/attendance/import-weekly` (또는 배치 스크립트) 구현.  
   - 시트 데이터 추출 → 임포트 → 전체 통계 화면으로 검수.

4. **(선택)** Google Sheets 연동이 필요하면, 시트 구조를 위 그리드와 맞춘 뒤 읽기 전용 연동 스펙을 추가.

## 6. 요약

- **한눈에 보기**: 그리드 = 행(그룹→다락방→이름), 열(주차), 셀(지각비/0/-) + 상단 지각비 기준·분기 납부.
- **데이터**: 현재 DB(`weekly_attendance`, `profiles`, `attendance_od_targets`, `late_fee_settlements`)만으로 구현 가능.
- **이관**: 이름+주차+지각비 엑셀/CSV → `user_id` 매핑 후 `weekly_attendance` upsert; 임포트 API 또는 배치로 진행.

이 문서를 기준으로 API·화면·임포트 순서로 구현하면 시트와 동일한 “전체 통계”를 시스템에서 재현할 수 있습니다.
