# 허브 앱스 (HUB Apps) 기획안

## 📱 기획 개요

허브 공동체를 더 가깝게 이어주고, 공동체의 일상적인 필요를 해결하는 다양한 앱들의 통합 플랫폼입니다.

### 기획 목적
- 공동체 구성원 간의 소통과 연결 강화
- 공동체 생활의 편의성 향상
- 공동체만의 특별한 경험 제공
- 디지털 공간에서의 공동체 문화 형성

---

## 🗓️ 1. 허브 캘린더 (HUB Calendar)

### 목적
공동체의 모든 일정을 한눈에 볼 수 있는 통합 캘린더 서비스

### 주요 기능
- **월간/주간/일간 뷰**: 다양한 캘린더 뷰 제공
- **카테고리별 필터링**: 
  - 예배/모임
  - 행사/이벤트
  - 아웃리치
  - 기타
- **일정 등록 및 수정**: 관리자 권한으로 일정 추가/수정/삭제
- **일정 상세 정보**:
  - 시간, 장소, 비고 
  - 준비물 및 안내사항
- **참가 신청**: 일정별 참가 신청 기능 (추후 연결 선택적)
- **반복 일정**: 주간/월간 반복 일정 설정

### 사용 시나리오
- "이번 주 예배 일정이 뭐지?"
- "다음 달 행사 일정을 미리 확인하고 싶어"

### 기술적 고려사항
- Supabase Calendar 테이블 설계
- 날짜/시간 필터링 최적화
- 캘린더 UI 라이브러리 (react-big-calendar 등)
- 알림 시스템 연동

### 관리자 페이지 기능
- **일정 관리** (`/admin/apps/calendar`):
  - 일정 추가/수정/삭제
  - 일정 목록 조회 (필터링: 카테고리, 날짜 범위)
  - 일정 상세 정보 관리
  - 반복 일정 설정 및 관리
  - 일정 활성화/비활성화
  - 참가 신청자 목록 조회 (추후 연결 시)
  - 일정 통계 (카테고리별, 월별 일정 수)
  - 일정 일괄 삭제/수정
  - 일정 템플릿 저장 및 재사용

## 🗺️ 2. 허브 맛집지도 (HUB Restaurant Map)

### 목적
공동체 구성원들이 추천하는 맛집을 공유하고, 함께 식사할 수 있는 장소를 찾는 서비스

### 주요 기능
- **맛집 등록 및 리뷰**:
  - 위치 (지도 연동)
  - 카테고리 (한식/양식/중식/일식/카페 등)

  

- **지도 뷰**: 
  - 카카오맵/구글맵 연동
  - 주변 맛집 검색
  - 거리 표시
- **필터링**:
  - 카테고리별
 

- **인기 맛집**: 공동체에서 가장 많이 추천받은 맛집 표시

### 사용 시나리오
- "오늘 점심 어디 갈까? 공동체에서 추천한 맛집 보자"
- "새로 발견한 맛집을 공동체에 공유하고 싶어"
- "이번 주 순모임 장소로 좋은 카페 찾기"

### 기술적 고려사항
- 지도 API 연동 (카카오맵/구글맵)
- 위치 기반 검색



### 관리자 페이지 기능
- **맛집 관리** (`/admin/apps/restaurant`):
  - 맛집 등록 승인/거부
  - 맛집 정보 수정/삭제
  - 맛집 목록 조회 (필터링: 카테고리)
  
  
  - 인기 맛집 관리 (추천 맛집 지정)
  
  - 맛집 검색 및 필터링

## 📖 3. 허브 용어사전 (HUB Glossary)

### 목적
공동체만의 특별한 용어와 개념을 정리하고 공유하는 사전 서비스

### 주요 기능
- **용어 등록 및 수정**:
  - 용어명
  - 카테고리 (신앙/공동체/행사/기타)
  - 정의 및 설명
  - 사용 예시
  - 관련 용어 링크
- **검색 기능**: 
  - 키워드 검색
  - 카테고리별 필터링
  - 초성 검색 지원
- **인기 용어**: 자주 검색되는 용어 표시
- **관련 용어 추천**: "이 용어와 함께 보면 좋은 용어"
- **용어 제안**: 구성원이 새로운 용어를 제안할 수 있는 기능
- **용어 히스토리**: 용어의 변천사 기록

### 사용 시나리오
- "순장이 '아웃리치'라고 했는데 정확히 뭔지 모르겠어"
- "새로 온 분들에게 공동체 용어를 설명해주고 싶어"
- "공동체에서 자주 쓰는 표현을 정리해두면 좋겠어"

### 기술적 고려사항
- 검색 최적화 (Full-text search)
- 태그 시스템
- 용어 간 관계 그래프
- 관리자 승인 시스템

### 관리자 페이지 기능
- **용어 관리** (`/admin/apps/glossary`):
  - 용어 등록/수정/삭제
  - 용어 제안 승인/거부
  - 용어 목록 조회 (필터링: 카테고리, 승인 상태)
  - 용어 검색 및 정렬
  - 관련 용어 연결 관리
  - 용어 히스토리 관리
  - 인기 용어 통계
  - 용어 카테고리 관리
  - 용어 일괄 수정/삭제
  - 용어 순서 조정 (order_index)

## 🙏 4. 허브 아웃리치 기도카드 게시판 (HUB Outreach Prayer Board)

### 목적
아웃리치와 선교를 위한 기도 제목을 공유하고 함께 기도하는 공간

### 주요 기능
- **기도 제목 등록**:
  - 제목
  - 상세 내용
  - 카테고리 (개인/가족/공동체/선교지/아웃리치 등)
  - 기도 기간 설정
  - 익명/실명 선택

- **기도 카운트**: 
  - "함께 기도해요" 버튼
  - 기도 횟수 표시
- **정렬 및 필터링**:
  - 최신순/인기순
  - 카테고리별
  - 기도 기간별
- **알림 기능**: 
  - 새로운 기도 제목 알림
  - 기도 응답 알림
- **기도 일기**: 개인 기도 일기 작성 (선택적)

### 사용 시나리오
- "이번 주 아웃리치를 위해 기도 제목을 올리고 싶어"
- "선교지의 기도 제목을 확인하고 함께 기도하고 싶어"
- "기도가 응답되어서 감사 나눔을 하고 싶어"

### 기술적 고려사항
- 익명 게시 기능
- 기도 카운트 실시간 업데이트
- 알림 시스템
- 민감한 정보 보호 (개인정보)

### 관리자 페이지 기능
- **기도 제목 관리** (`/admin/apps/prayer`):
  - 기도 제목 조회 (필터링: 카테고리, 기도 기간, 상태)
  - 부적절한 기도 제목 삭제/수정
  - 기도 제목 공개/비공개 설정
  - 기도 응답 기록 관리
  - 기도 통계 (카테고리별, 기간별 기도 제목 수)
  - 기도 카운트 현황 조회
  - 기도 제목 검색
  - 기도 제목 일괄 관리
  - 기도 제목 우선순위 설정 (고정 게시)
  - 기도 제목 알림 설정 관리

## ✅ 5. 허브 체크인 (HUB Check-in)

### 목적
간편하고 빠른 출석 체크 시스템

### 주요 기능
- **QR 코드 체크인**:
  - 행사/모임별 고유 QR 코드 생성
  - 스마트폰으로 간편 스캔
  - 자동 시간 기록
- **수동 체크인**: 
  - 이름 검색으로 체크인
  - 관리자 대리 체크인
- **출석 통계**:
  - 개인 출석 현황
  - 행사별 출석률
  - 월간/연간 출석 통계
- **출석 이력 조회**: 
  - 개인 출석 이력
  - 행사별 출석자 목록
- **알림 기능**: 
  - 체크인 시간 알림
  - 출석 확인 요청
- **출석 리포트**: 
  - 주간/월간 출석 리포트 생성
  - 엑셀 다운로드

### 사용 시나리오
- "예배 시작 전 빠르게 체크인하고 싶어"
- "이번 달 출석 현황을 확인하고 싶어"
- "셀 모임 출석자를 확인해야 해"

### 기술적 고려사항
- QR 코드 생성 라이브러리
- 실시간 출석 현황 업데이트
- 오프라인 체크인 지원 (선택적)
- 개인정보 보호 (출석 정보)

### 관리자 페이지 기능
- **체크인 관리** (`/admin/apps/checkin`):
  - 행사/모임 등록 및 관리
  - QR 코드 생성 및 관리
  - 체크인 세션 생성/종료
  - 실시간 출석 현황 모니터링
  - 출석자 목록 조회 (행사별, 사용자별)
  - 수동 체크인/체크아웃
  - 출석 통계 (행사별 출석률, 개인 출석률)
  - 출석 리포트 생성 및 다운로드 (엑셀)
  - 출석 이력 조회 및 검색
  - 출석 데이터 수정/삭제
  - 출석 알림 설정
  - 출석 패턴 분석 (월별, 연도별 트렌드)

## ⏱️ 6. 허브 기도 시간 (HUB Prayer Time)

### 목적
노크 기도 톡방에서 착안한 기도 시간 자동 측정 및 기록 시스템 (열품타 스타일)

### 주요 기능
- **기도 시간 측정**:
  - 기도 시작 버튼 클릭 시 자동 시간 기록 시작
  - 기도 종료 버튼 클릭 시 자동 시간 기록 종료
  - 실시간 타이머 표시
  - 초기화 기능
- **개인 기도 시간 통계**:
  - 오늘 나의 기도 시간
  - 나의 총 기도 시간
  - 날짜별 기도 시간 조회
  - 타임라인에 기도한 만큼 색 진하게 표시 (선택적)
- **공동체 기도 시간**:
  - 허브 전체 기도 시간 합계
  - 각 허브 지체의 기도 시간 목록 (이름, 시간 표시)
  - 실시간 기도 중인 사람 목록 (LIVE 표시)
- **독려 기능**:
  - 서로의 기도 시간 확인
  - 기도 시간 순위 (선택적)
  - 기도 시간 목표 설정 (선택적)

### 사용 시나리오
- "기도 시작할 때 버튼 누르고, 마칠 때 버튼 눌러서 자동으로 기록되게 하고 싶어"
- "오늘 내가 기도한 시간을 확인하고 싶어"
- "다른 허브 지체들의 기도 시간을 보고 서로 독려하고 싶어"
- "허브 전체 기도 시간을 확인하고 싶어"

### 기술적 고려사항
- 실시간 타이머 (클라이언트 측)
- 기도 시간 자동 계산 및 저장
- 날짜별 집계 쿼리 최적화
- 실시간 기도 중인 사람 목록 (WebSocket 또는 폴링)
- 타임라인 시각화 (캘린더 형태)

### 관리자 페이지 기능
- **기도 시간 관리** (`/admin/apps/prayer-time`):
  - 전체 사용자 기도 시간 조회
  - 날짜별/사용자별 기도 시간 통계
  - 기도 시간 데이터 수정/삭제 (이상치 처리)
  - 기도 시간 리포트 생성
  - 기도 시간 목표 설정 및 관리
  - 기도 시간 통계 대시보드

## 🔍 7. 분실물 (HUB Lost & Found)

### 목적
공동체 내에서 그날 하루 분실물을 사진으로 공유하고 찾는 서비스. **인스타그램 피드 형태**로 운영.

### 컨셉 (인스타그램 스타일)
- **관리자만 등록**: 관리자가 그날 발견된 분실물을 한 장(또는 최대 두 장)의 사진으로 업로드
- **한 포스트 = 그날 하루 분실물**: 사진 한 장에 그날의 분실물이 모두 담김 (1~2장/일)
- **피드 UI**: 사용자는 인스타그램처럼 스크롤하며 날짜별 분실물 사진 확인
- **클릭 시 상세**: 포스트 클릭 시 해당 날짜, 사진 확대, 간단 설명(선택) 표시

### 주요 기능
- **관리자 – 분실물 포스트 등록**:
  - 그날 날짜 선택
  - 사진 1~2장 업로드 (Cloudinary)
  - 선택: 간단 메모 (예: "본당 앞 테이블")
- **사용자 – 피드 보기**:
  - 인스타그램 그리드/피드 형태로 날짜순(최신순) 표시
  - 썸네일 클릭 시 모달/상세에서 사진 확대 + 날짜 + 메모
  - 날짜별 필터(선택)
- **이미지 저장**: **Cloudinary** 사용 (업로드, 리사이즈, CDN 배달)

### 사용 시나리오
- "일요일 예배 후 잃어버린 것 같은데, 그날 분실물 사진만 봐도 될 것 같아"
- "관리자가 오늘 분실물 테이블 사진 올렸나 확인해볼게"
- "지난주 분실물 사진 두 장 보고 내 지갑 있는지 확인했어"

### 기술적 고려사항
- **Cloudinary**: 이미지 업로드, 변환(썸네일/원본), CDN URL 저장
- **폴더 분리**: 사진부스와 같은 Cloudinary 계정을 쓰되, 분실물은 전용 폴더 사용 (기본값 `hub/lost-found`). env `CLOUDINARY_LOST_FOUND_FOLDER`로 변경 가능.
- 피드용 썸네일 크기와 상세용 원본(또는 large) 구분
- 포스트 데이터: 날짜, Cloudinary public_id 또는 URL 1~2개, 메모(선택), 생성일

### 관리자 페이지 기능
- **분실물 관리** (`/admin/apps/lost-found`):
  - 분실물 포스트 목록 (날짜순)
  - **새 포스트 등록**: 날짜 선택 + 사진 1~2장 업로드(Cloudinary) + 메모(선택)
  - 포스트 수정/삭제 (사진, 날짜, 메모)
  - Cloudinary: `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET` (사진부스와 동일 키 사용 가능). 분실물 전용 폴더: `CLOUDINARY_LOST_FOUND_FOLDER` (선택, 기본 `hub/lost-found`)
